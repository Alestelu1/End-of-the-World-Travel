# AI Instructions

**Project:** End of the World Travel

**Version:** 1.0

**Status:** AI Collaboration Guide

---

# Purpose

This document defines how AI assistants should contribute to End of the World Travel.

Every AI working on this repository must follow these instructions before proposing, modifying, or generating any content or code.

The objective is consistency, reliability, and long-term maintainability.

# Project Identity

End of the World Travel is:

An independent editorial publication.

A documentary project.

Focused on Southern Chile.

Focused on geography, history, culture, conservation, and responsible travel.

It is NOT:

A travel agency.

A booking platform.

A mass tourism website.

An influencer blog.

A content farm.

---

# Editorial Rules

Always:

Write in third person.

Maintain a calm, documentary tone.

Prioritize facts.

Provide context.

Respect local communities.

Respect protected areas.

Respect Indigenous peoples.

Avoid marketing language.

---

# Technical Rules

Prefer:

Semantic HTML

Modern CSS

Minimal JavaScript

Fast loading

Accessible code

Reusable components

Avoid unnecessary dependencies.

Do not introduce frameworks without approval.

---

# Source Rules

Prioritize:

Official Chilean institutions

Academic sources

Museums

Scientific publications

Conservation organizations

Never invent information.

Verify uncertain facts.

Do not cite AI systems as sources.

---

# Photography Rules

Use only real photographs.

Never replace documentary photographs with AI-generated images.

Every published image must include proper attribution.

AI images may only be used for:

Internal mockups

Design concepts

Planning

They must never appear as factual documentation.

---

# Design Rules

Respect the Design System.

Do not redesign components unnecessarily.

Maintain visual consistency.

Do not introduce trendy UI elements that conflict with the editorial style.

---

# SEO Rules

Think in entities, not keywords.

Write for readers.

Maintain semantic internal linking.

Use descriptive headings.

Avoid keyword stuffing.

---

# Code Changes

Before editing:

Understand the existing structure.

Reuse existing components.

Avoid duplicate code.

Preserve accessibility.

Explain significant architectural changes.

---

# Content Generation

Never invent:

Historical facts

Statistics

Coordinates

Dates

Official designations

Legal information

When information is missing:

State that verification is needed.

Suggest official sources.

---

# Puerto Williams Rule

Whenever appropriate, use the editorial context:

Puerto Williams was recognized as a city by Chile's National Statistics Institute (INE) in 2019 and is presented in Chilean official sources as the southernmost city on Earth.

Puerto Toro should be described separately as Chile's southernmost permanently inhabited settlement.

---

# Image Credits

Do not remove image credits.

If a new photograph is added:

Update:

PHOTO_CREDITS.md

photo-credits.html

Include the source and licensing information.

---

# Git Workflow

When proposing code changes:

Explain the purpose.

Show the expected result.

Minimize unrelated edits.

Keep commits focused on a single topic whenever possible.

---

# Performance

Protect performance.

Avoid unnecessary scripts.

Optimize images.

Prefer static rendering.

Keep the site lightweight.

---

# Communication Style

When suggesting changes:

Explain why.

Present alternatives when appropriate.

Do not overwrite existing work without approval.

If uncertain:

Ask.

Do not assume.

---

# Long-Term Vision

Every contribution should help transform End of the World Travel into a trusted editorial reference for Southern Chile.

The project values:

Accuracy

Transparency

Editorial independence

Respect

Maintainability

Scalability

Quality over speed.

---

# Final Principle

If a proposed change improves performance, accessibility, editorial quality, factual accuracy, or maintainability without compromising the project's principles, it is probably the right direction.

If a proposed change prioritizes appearance, trends, or speed at the expense of trust, it should not be implemented.
