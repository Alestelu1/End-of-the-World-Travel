# Contributing

**Project:** End of the World Travel

**Version:** 1.0

**Status:** Contributor Guide

---

# Welcome

Thank you for your interest in contributing to End of the World Travel.

This project documents the geography, history, culture, natural heritage, and communities of Southern Chile through an independent editorial approach.

Every contribution should help improve the quality, accuracy, and usefulness of the publication.

---

# Before You Contribute

Please read the documentation inside the `/docs` directory before proposing changes.

Required reading:

* design-system.md
* editorial-style-guide.md
* sources-policy.md
* photography.md
* content-roadmap.md
* seo-guidelines.md
* architecture.md

These documents define the standards of the project.

---

# Editorial Principles

Every contribution should be:

Accurate

Respectful

Well documented

Neutral

Educational

Transparent

Avoid promotional language.

Avoid sensationalism.

Avoid unsupported claims.

---

# Code Contributions

When modifying code:

Prefer semantic HTML.

Keep CSS organized.

Use minimal JavaScript.

Avoid unnecessary dependencies.

Preserve accessibility.

Reuse existing components whenever possible.

Explain significant architectural changes.

---

# Content Contributions

New articles should:

Be based on reliable sources.

Provide historical and geographic context.

Respect Indigenous communities.

Respect protected areas.

Use official place names.

Avoid duplication.

When information is uncertain, identify it clearly and verify before publication.

---

# Sources

Always prioritize:

Official Chilean institutions

Scientific publications

Museums

Universities

Regional governments

Protected area authorities

Every factual statement should be verifiable.

---

# Photography

Only use photographs with clear licensing or permission.

Every image must include attribution.

AI-generated images are acceptable only for internal design concepts and planning.

They must never be published as documentary photographs.

---

# Pull Requests

Keep pull requests focused.

One topic per pull request whenever possible.

Include:

Purpose

Summary of changes

Files modified

Reasoning

Testing performed

Avoid unrelated changes in the same submission.

---

# Commit Messages

Use clear and descriptive commit messages.

Examples:

Add Puerto Toro destination guide

Improve Home editorial hierarchy

Update photo credits

Refactor destination cards

Avoid vague messages such as:

Update

Fix

Changes

Miscellaneous

---

# Respect the Project

The goal is not rapid expansion.

The goal is long-term quality.

If a change makes the publication:

More accurate

More readable

Better documented

More accessible

More maintainable

it is probably a good contribution.

---

# AI Contributions

AI assistants are welcome to assist with:

Writing drafts

Code refactoring

Documentation

Accessibility improvements

SEO suggestions

Image optimization

Internal linking

However, AI should never:

Invent facts

Fabricate statistics

Create fictional historical events

Replace documentary photographs with AI-generated images

Override established editorial decisions

When uncertain, ask for clarification.

---

# Review Process

Every significant contribution should be reviewed before publication.

Editorial quality is more important than publication speed.

Constructive discussion is encouraged.

---

Developers

# Final Principle

Every contribution should leave the project in a better state than it was before.

Preserve simplicity.

Preserve trust.

Preserve editorial independence.

Respect Southern Chile, its communities, and its natural heritage.
