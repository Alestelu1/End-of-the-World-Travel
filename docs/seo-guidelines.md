# SEO Guidelines

**Project:** End of the World Travel

**Version:** 1.0

**Status:** Active SEO Standard

---

# Purpose

SEO should support readers first and search engines second.

The objective is to help people discover reliable information about Southern Chile while building a structured knowledge base around geography, history, culture and natural heritage.

SEO should improve understanding, not manipulate rankings.

---

# SEO Philosophy

Write for humans.

Structure for search engines.

Respect editorial quality.

Never sacrifice factual accuracy for rankings.

---

# Core Topics

The publication focuses on:

* Southern Chile
* Magallanes Region
* Chilean Antarctic Region
* Patagonia
* Tierra del Fuego
* Strait of Magellan
* Beagle Channel
* Cape Horn
* Puerto Williams
* Puerto Toro
* Punta Arenas
* Antarctica gateways

---

# Entity-Based SEO

Every page should represent one primary entity.

Examples:

Puerto Williams

Cape Horn

Beagle Channel

Tierra del Fuego

Strait of Magellan

Yendegaia National Park

Carretera Austral

Ruta Vicuña–Yendegaia

Each entity should connect naturally to related places.

---

# Internal Linking

Every article should link to:

Nearby destinations

Related routes

Protected areas

Museums

Communities

Editorial guides

Relevant experiences

Sources & Methodology

Links should help readers understand relationships, not simply increase page views.

---

# URL Structure

Use simple descriptive URLs.

Examples:

/puerto-williams

/cape-horn

/tierra-del-fuego

/beagle-channel

/experiences

/journal

/about

/sources

Avoid unnecessary folders.

Keep URLs stable.

---

# Titles

Titles should be descriptive.

Examples:

Puerto Williams Travel Guide

Cape Horn

Strait of Magellan

Yendegaia National Park

Avoid clickbait.

Avoid unnecessary punctuation.

---

# Meta Descriptions

Write naturally.

Summarize the page.

Avoid keyword repetition.

Keep descriptions factual.

---

# Heading Structure

Use semantic hierarchy.

H1

One per page.

H2

Major sections.

H3

Subsections.

Never skip heading levels without reason.

---

# Structured Data

Use Schema.org where appropriate.

Possible schemas:

WebSite

Organization

Article

Place

TouristDestination

ImageObject

BreadcrumbList

FAQ (only when genuinely useful)

Do not add structured data that does not reflect the page.

---

# Images

Every image should include:

Meaningful filename

Descriptive alt text

Credits

Lazy loading

Optimized format

---

# Performance

Target:

Fast loading

Minimal JavaScript

Optimized images

Static pages whenever possible

Semantic HTML

Accessibility compliant

---

# Geographic Consistency

Use official place names.

Examples:

Puerto Williams

Cape Horn (Cabo de Hornos)

Beagle Channel (Canal Beagle)

Strait of Magellan (Estrecho de Magallanes)

Include Spanish names where useful for clarity.

---

# Content Quality

Every page should answer:

Where is it?

Why is it important?

What is its historical context?

How does it relate to Southern Chile?

Where can readers find official information?

Avoid shallow content.

---

# Keyword Philosophy

Do not write for keywords alone.

Primary keywords should emerge naturally from the topic.

Prioritize entities and context over keyword density.
